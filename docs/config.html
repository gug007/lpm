Configuration

Everything you can put in a project config file.

Project

Top-level fields that identify the project. 

name: myapp
root: ~/Projects/myapp
Field Type Required Description
name string yes Project name (used as identifier)
root string yes Project root directory. Supports ~.

Services

Long-running processes that lpm starts and stops together. At least one service is required. 

services:
  # shorthand — just the command
  worker: celery -A backend worker

  # full form
  api:
    cmd: go run .
    cwd: ./backend            # working directory (relative to root)
    port: 8080                # port (0-65535, must be unique)
    env:                      # environment variables
      DATABASE_URL: postgres://localhost/myapp
Field Type Required Description
cmd string yes Shell command to run
cwd string no Working directory (relative to root or absolute). Supports ~.
port int no Port the service listens on (0-65535). Must be unique across services.
env map no Environment variables
profiles []string no Profiles this service belongs to

Actions

One-shot commands like test runners, migrations, or deploy scripts. Run from the app or CLI with lpm run <project> <action>. 

actions:
  test: go test ./...         # shorthand

  migrate:                    # full form
    cmd: rails db:migrate
    cwd: ./backend
    label: Run Migrations     # display name in the UI
    confirm: true             # ask before running
    display: button           # show as a button instead of in menu
    env:
      RAILS_ENV: production
Field Type Required Description
cmd string yes Shell command to run
label string no Display name in the UI
cwd string no Working directory. Supports ~.
env map no Environment variables
confirm bool no Prompt for confirmation before running
display string no button or menu (default)

Terminals

Persistent interactive shells you can open from the app. 

terminals:
  console: rails console     # shorthand

  psql:                       # full form
    cmd: psql myapp_dev
    label: Database
    cwd: ./backend
    display: button
Field Type Required Description
cmd string yes Shell command to run
label string no Display name in the UI
cwd string no Working directory. Supports ~.
env map no Environment variables
display string no button or menu (default)

Profiles

Named subsets of services. Start a profile with lpm myapp -p <profile> or pick one from the desktop app. If no profile is specified, all services start. 

profiles:
  frontend-only:
    - frontend
  full-stack:
    - api
    - frontend
    - worker

Each service name must reference a service defined in services.

Global Config

~/.lpm/global.yml defines actions and terminals shared across all projects. Project-level entries take precedence when names collide.

~/.lpm/global.yml 
actions:
  docker-prune:
    cmd: docker system prune -f
    label: Docker Prune
    confirm: true

terminals:
  htop: htop

Global config only supports actions and terminals — not services or profiles.

Path resolution

  • ~ expands to your home directory
  • Relative cwd paths resolve relative to root
  • Absolute paths are used as-is

Validation

Config is validated on load and save. Validation checks:

  • At least one service is defined
  • All cmd fields are non-empty
  • Ports are in range 0-65535 with no duplicates
  • All cwd paths point to existing directories
  • Profile entries reference existing services
